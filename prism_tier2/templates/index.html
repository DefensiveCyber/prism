Cleaning Rules
Control how PRISM strips dirty or non-conforming content before routing files to landing zones
How It Works
⌁ line
Each line is one event. Lines matching the Line Filter regex are kept. Everything else — banners, headers, separators — goes to a .noise.log sidecar file alongside the clean output in the landing zone.
Examples: Cisco ASA, PAN CSV, Linux syslog, CEF, LEEF, auditd
⌁ multiline
Events span multiple lines. PRISM extracts complete event blocks and strips everything outside them. Six sub-modes: json_lines, json_object, xml_event, zeek_tsv, iis, csv_with_header.
Examples: WinEvent XML, CloudTrail JSON, Suricata EVE, Zeek TSV
⌁ passthrough
File is routed as-is. No cleaning applied. Use when no reliable filter exists.
⌁ auto
PRISM derives the mode automatically from the signature's required_patterns at classification time. No manual config needed. This is the default for all signatures.
Override by setting an explicit Filter Mode in the signature.
💡 Noise files — stripped lines are never deleted permanently. They are written to filename.noise.log in the same landing zone folder as the clean file. You can review, reprocess, or delete them at any time.
